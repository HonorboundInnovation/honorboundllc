An agentic development harness that actually works inside the repo.

Vegvisir is not a floating chat box with a motivational sticker. It is a local-first software workbench where an AI agent can inspect files, edit code, run bounded commands, use durable project memory, coordinate subagents, verify results, and keep the transcript honest.

What Vegvisir is

A command center for model-assisted software work.

Vegvisir wraps a model provider in a real engineering runtime. The assistant is given an active workspace, a tool registry, a policy boundary, project memory, and a transcript. That means it can do practical work instead of merely describing work.

The harness is designed around the loop that matters: orient from evidence, plan the next move, execute with tools, verify the result, and report what changed. It is built for repository maintenance, feature work, debugging, documentation, operations, migrations, and security-aware review.

Runtime model

The model thinks. Vegvisir gives it hands, memory, and rules.

Workspace-scoped tools

02

Policy-aware execution

03

Verification-first workflow

Vegvisir is built to run checks before claiming success: tests, builds, render passes, git status, diff review, and targeted diagnostics.

Working loop

Every serious answer should have a trail.

Vegvisir surfaces tool activity, file changes, command results, subagent updates, and verification status so the session does not collapse into “trust me, bro.” The transcript becomes the operational record.

Memory and context

CMS remembers. ECM decides what enters the room.

CMS-v2 durable memory

Project facts, decisions, repo roots, user preferences, and task continuity can be stored outside the model context window so important details survive the next session.

ECM active context exposure

Vegvisir does not dump the entire attic into every prompt. ECM prepares the relevant working context for the current turn while respecting budget and scope.

HBSE secret boundary

Credentials belong behind brokered secret references, not plaintext chat. Vegvisir’s design keeps auth handling separate from model-visible memory and conversation text.

Subagents

Parallel review without losing the evidence.

  1. 01

    Bounded delegation

    Child agents can be assigned narrow goals, file scopes, budgets, allowed tools, and maximum steps for reconnaissance, testing, documentation review, or design critique.

  2. 02

    Board-backed coordination

    Subagent records are written to a durable board with status, goal, workspace, file scope, budget, checkpoint, final answer, and error state.

  3. 03

    Transcript visibility

    Subagent updates are mirrored into the parent transcript, including captured output, so delegated work does not vanish into a side channel. No missing little gremlin footprints.
